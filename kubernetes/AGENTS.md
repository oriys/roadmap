# Repository Guidelines

## Project Structure & Module Organization
- `src/` React + TypeScript app (Vite). Entry `src/main.tsx`; main UI and roadmap content live in `src/App.tsx`.
- `src/components/ui/` reusable UI primitives (Radix + Tailwind).
- `src/lib/` data/content helpers (`lesson-quizzes.ts`, `doc-questions.ts`, `utils.ts`).
- `public/` static assets served as-is.
- `dist/` production build output (generated by `pnpm build`; don’t hand-edit).

## Build, Test, and Development Commands
This project uses `pnpm` (keep `pnpm-lock.yaml` up to date).
- `pnpm install`: install dependencies.
- `pnpm dev`: start the local dev server.
- `pnpm build`: run `tsc` (typecheck) and build to `dist/`.
- `pnpm preview`: serve the built app from `dist/`.
Runtime: Vite 7 requires Node `^20.19.0 || >=22.12.0`.

## Coding Style & Naming Conventions
- TypeScript is `strict` (see `tsconfig.json`); avoid unused locals/params and keep types accurate.
- Match existing formatting: 2-space indentation, double quotes, no semicolons.
- Prefer the `@/` alias for `src/*` imports (configured in `vite.config.ts`).
- File naming: `src/components/ui/*` uses kebab-case (e.g. `radio-group.tsx`); React components export PascalCase symbols.

## Testing Guidelines
No automated test runner is configured yet. Validate changes with `pnpm build` and a quick smoke test in `pnpm dev` (or `pnpm preview` for production output).

## Commit & Pull Request Guidelines
There’s no established Git history yet—use clear, conventional messages like `feat: add week 2 quiz` or `fix: correct resource link`.
PRs should include: summary, test steps (`pnpm dev` / `pnpm build`), and screenshots for UI changes. Don’t commit `node_modules/`; only commit `dist/` if you intentionally ship static output.

## Security & Configuration Tips
- If you add env vars, use Vite’s client-side prefix (`VITE_*`) and keep secrets out of the repo (prefer `.env.local`).

## Content Editing Tips
- `lesson_quizzes.md` is parsed by `src/lib/doc-questions.ts`; keep headings like `## w1-1` and numbered lines like `1. ...`.
- Quiz IDs in `src/lib/lesson-quizzes.ts` follow `<lessonId>-q<number>` (e.g. `w1-1-q1`); `answer` is a 0-based option index.
